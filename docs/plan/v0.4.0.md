## §I — Interfaces

### SSE event types

```
DashboardEvent (kebab-case JSON, sent via Event::default().event(name).json_data(payload)):
  SkillInstalled      { id, name }                     → event: skill-installed
  SkillUninstalled    { id }                            → event: skill-uninstalled
  SkillToggled        { id, enabled }                   → event: skill-toggled
  SkillSyncSucceeded  { count }                         → event: skill-sync-succeeded
  SkillSyncFailed     { count, error }                  → event: skill-sync-failed
  ConfigChanged       {}                                → event: config-changed
  ProviderRegistered  { id }                            → event: provider-registered
  SecretDeleted       { cred_id }                       → event: secret-deleted
  SecretAdded         { cred_id, provider? }            → event: secret-added
  CatalogSourceAdded  { name }                          → event: catalog-source-added
  CatalogSourceRemoved{ name }                          → event: catalog-source-removed
```

Special: `lag-detected` { count } emitted by Hub when a subscriber falls
behind; consumer treats as catch-up signal (blanket invalidate).

### SSE endpoint

`GET /api/events` — stream of `event:` + `data:` SSE frames.
Auth: `Authorization: Bearer <jwt>` (Next.js proxy) preferred; `?token=<jwt>`
fallback for raw EventSource clients.
Server: `LazyLock<broadcast::Sender<DashboardEvent>>` cap 128, mapped to
`Event::default().event(name).json_data(payload)`.

### Next.js proxy

`GET /api/events` route handler:
- `export const dynamic = "force-dynamic"` (prevents Next.js buffering)
- `auth()` → session
- `fetch(HUB_URL/api/events, { cache: "no-store", signal: request.signal })`
- `return new Response(upstream.body, headers)` (text/event-stream passthrough)

### Event → Query key map (frontend)

event|touched keys
skill-installed|skillKeys.all, catalogKeys.all
skill-uninstalled|skillKeys.all, catalogKeys.all
skill-toggled|skillKeys.all, catalogKeys.all
skill-sync-succeeded|syncKeys.all, skillKeys.all
skill-sync-failed|syncKeys.all, skillKeys.all
config-changed|settingsKeys.all
provider-registered|secretsKeys.all, agentKeys.all
secret-deleted|secretsKeys.all, agentKeys.all
secret-added|secretsKeys.all, agentKeys.all
catalog-source-added|catalogKeys.all
catalog-source-removed|catalogKeys.all
lag-detected|(all active queries)
onopen (initial OR reconnect)|(all active queries)

### Client architecture

```
EventSource("/api/events") (same-origin, proxy attaches Bearer)
  → EventBusService (globalThis singleton, HMR-safe)
    → onopen → queryClient.invalidateQueries()       (catch up)
    → addEventListener(name) → targeted invalidate  (live update)
    → useEventStore.setState(...)                     (sidebar indicator)
```

Connection states: `"connecting" | "connected" | "reconnecting" | "disconnected"`.
`connecting` initial; `reconnecting` after first `onopen` if `onerror` fires;
`disconnected` when `EventSource.readyState === CLOSED` (server told us to stop).
EventSource auto-reconnects natively (~3s default) — no custom backoff.
